# 코드 리뷰 - 8x8 도트매트릭스 조이스틱 미로 게임

신규/수정 파일: `shiftreg.c`, `dotmatrix.c`, `adc.c`, `joystick.c`, `main.c`, `device_driver.h`, `exception.c`

## `shiftreg.c` (74HC595 드라이버)

- 로직 자체는 정상 (MSB부터 8비트 시프트 후 RCLK 래치).
- **사소한 점**: `Macro_Set_Bit`/`Clear_Bit` 사이에 지연이 전혀 없음. STM32F411이 최대 100MHz라 SER→SRCLK 사이 셋업타임이 이론상 아주 빠듯할 수 있으나, 실제 보드에서 문제없이 동작 확인됨. 배선이 길어지거나 노이즈가 생기면 짧은 지연(`for(i=0;i<2;i++);` 등) 추가 고려.

## `dotmatrix.c` (프레임버퍼 + 스캔)

- Row/Col 극성 반전(`~data`, Row Active High)은 실측으로 검증 완료, 정확함.
- **레이스 컨디션 (경미)**: `Dot_Matrix_Set_Player(row, col)`가 `Player_Row`, `Player_Col`을 두 번의 별도 대입으로 갱신. 그 사이 `TIM4` 스캔 ISR이 끼어들면 한 프레임(약 16ms) 동안 row는 새 값, col은 이전 값인 상태로 읽힐 수 있음. 눈에 안 보이는 수준이나 엄밀히는 비원자적.
- **사소한 점**: `Dot_Matrix_Init()` 직후 첫 프레임에서 스캔 순서상 Row0이 건너뛰어져(0→1로 시작) 딱 한 프레임만 Row0이 안 켜짐. 실질적 문제 아님.
- **죽은 코드**: `Dot_Matrix_Clear()`, `Dot_Matrix_Set_Pixel()`이 `main.c` 어디서도 호출되지 않음(벽은 `Load`로, 플레이어는 `Set_Player`로만 관리). 안 쓸 거면 정리 대상.

## `adc.c` / `joystick.c`

- 채널 전환(`ADC1_Select_Channel`) → 시작 → 폴링 → 읽기 흐름이 기존 1501 랩 패턴과 동일, 신뢰도 높음.
- SW 버튼은 풀업 + active-low로 올바르게 처리. 하드웨어 디바운스 커패시터는 없지만 `main.c`의 `while(Joystick_Get_SW());` + `TIM2_Delay(200)` 조합으로 소프트웨어적으로 충분히 커버됨.
- 발견된 버그 없음.

## `main.c` (미로 로직)

- 미로 생성 알고리즘은 이미 실제 버그(도착점이 벽으로 막히는 경우, 시뮬레이션 결과 약 17% 확률)를 발견해서 재시도 로직으로 수정 완료. 최대 5회 재시도면 충분함을 시뮬레이션으로 검증함. 현재 상태 정상.
- **`srand` 시드 엔트로피 낮음**: `srand(center_x ^ (center_y << 16))`인데, 실측 결과 조이스틱 중립값이 부팅마다 X≈3083~3099, Y≈3015~3027로 변동폭이 작음. 전원을 껐다 켤 때마다 **첫 번째 미로가 비슷하게 나올 가능성** 있음(이후 미로들은 `rand()` 시퀀스가 이어져서 정상적으로 다양해짐). 데모용으론 무해하나, 신경 쓰이면 `SysTick_Get_Time()`을 같이 섞거나 ADC를 여러 번 읽어 하위 비트를 누적하는 식으로 엔트로피 보강 가능.
- **설계상 특성(버그 아님) - 대각선 입력 시 X축 우선**: `if(x...) else if(x...) else if(y...) else if(y...)` 구조라 조이스틱을 대각선으로 기울이면 좌우 이동이 항상 상하보다 우선 처리됨.
- **설계상 특성 - 길게 누르면 자동 반복 이동**: 한 번 이동 후 300ms가 지나도 여전히 임계값을 넘는 상태면 중앙으로 안 돌아와도 바로 다음 칸으로 이동함(놓았다 다시 기울여야 하는 방식이 아님). 원하면 "중앙으로 돌아와야 다음 이동 허용" 방식으로 변경 가능.
- **이론적 미해결 경우**: 재시도 50회가 전부 실패하면(확률상 0.17^50 수준, 사실상 불가능) 그 판만 도착점이 막힌 채로 남음.

## `device_driver.h`, `exception.c`

- extern 선언과 실제 구현이 정확히 일치함.
- `TIM4_IRQHandler`에서 pending clear 이후에 `Dot_Matrix_Scan_ISR()`를 호출하는 순서도 올바름.

## 종합 평가

치명적인 버그는 없음(있었던 진짜 버그 - 도착점 미로 생성 실패 - 는 이미 수정함). 나머지는 전부 "언젠가 신경 쓰일 수도 있는" 수준의 사소한 항목.

우선순위:
1. (선택) `srand` 엔트로피 개선
2. (선택) `Set_Player` 레이스 컨디션 - 필요하면 갱신 중 인터럽트 잠깐 비활성화
3. (선택) 이동 반복 방식 (누르고 있으면 계속 vs 놓았다 다시 기울여야 함)
4. (정리) 안 쓰는 `Dot_Matrix_Clear`/`Set_Pixel` 제거 또는 유지 결정
