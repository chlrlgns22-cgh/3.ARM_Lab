# 8x8 도트매트릭스 조이스틱 미로 게임

STM32F411 + 1088BS(8x8 도트매트릭스) + 74HC595(시프트 레지스터) + HW-504(조이스틱)로 만든
미니 미로 게임. 조이스틱으로 점을 움직여서 왼쪽 아래(시작)에서 오른쪽 위(도착)까지 가는 게임이며,
도착할 때마다 미로가 랜덤으로 새로 생성됨.

## 부품 목록

| 부품 | 비고 |
|---|---|
| STM32F411 보드 | 이 프로젝트(Project1) 기준 |
| 1088BS 8x8 도트매트릭스 | 공통 음극(BS) 타입 |
| 74HC595 시프트 레지스터 | 8x8 매트릭스 Col 구동용 |
| HW-504 조이스틱 모듈 | X/Y 아날로그 + SW 버튼 |
| 220~330Ω 저항 x8 | 74HC595 출력 ~ 매트릭스 Col 사이 전류 제한 |
| 패시브 부저 | 충돌음/성공음 |

## 배선표

| 핀 | 연결 대상 | 비고 |
|---|---|---|
| PA6 | 조이스틱 VRx | ADC1_IN6 |
| PA7 | 조이스틱 VRy | ADC1_IN7 |
| PC8 | 조이스틱 SW | 내부 풀업, 누르면 GND |
| PB0 | 부저 신호선 | TIM3 PWM |
| PB1 | 74HC595 DS(SER) | |
| PB2 | 74HC595 SHCP(SRCLK) | |
| PB3 | 74HC595 STCP(RCLK) | |
| PC0~PC7 | 1088BS Row1~Row8 | **Active High** (실측 결과 반영) |
| 74HC595 Q0~Q7 | 1088BS Col1~Col8 | **Active Low**, 저항 경유 (실측 결과 반영) |

> Row/Col 극성은 처음 가정(Row Low켜짐/Col High켜짐)과 실제가 반대였음 — 배선 후 정적 테스트(Row0+Col0만 켜서 확인)로 찾아내서 `dotmatrix.c`에서 반전 처리함.

## 조작법

- 조이스틱 기울이기 : 점이 그 방향으로 한 칸 이동 (벽이면 이동 취소 + 충돌음)
- 조이스틱 버튼(SW) 누르기 : 현재 미로에서 시작점으로 즉시 리셋
- 도착(오른쪽 위)하면 성공음이 나고, 자동으로 새 미로가 생성되어 다시 시작점에서 진행

## 동작 원리

- **디스플레이 갱신** : TIM4 인터럽트(2ms마다) 로 Row를 한 줄씩 순차 점등(멀티플렉싱), 8줄x2ms = 16ms(약 62.5Hz)로 전체 화면 갱신
- **플레이어 표시** : 벽(배경) 프레임과 별도 레이어로 관리되며, 일정 주기로 점멸시켜 벽과 구분되게 함
- **미로 생성** : 시작점에서 재귀 백트래킹으로 무작위로 길을 뚫어나가는 방식. 통로끼리 붙어 지름길이 생기지 않도록 "새로 열 칸의 이미 열린 이웃이 1개(지금 오는 방향)뿐"인 경우만 허용. 도착점이 열리는 순간 멈춤.
  - 도착점이 모서리라 이웃이 2개뿐이라 드물게(시뮬레이션 결과 약 17%) 다른 통로에 둘러싸여 영영 못 뚫리는 경우가 있어서, 도착점이 열릴 때까지 자동 재시도하도록 처리함(평균 1~2회, 최대 몇 회면 충분).
- **조이스틱 보정** : 부팅 시점(안 건드린 상태)의 X/Y 값을 중립점으로 자동 측정해서 저장 — 개체별로 중립점이 중앙(2048)이 아니라 치우쳐 있는 경우가 많아서(실측 X≈3090, Y≈3020) 절대값이 아닌 "중립점 대비 얼마나 벗어났는지"로 판단.

## 파일 구성

| 파일 | 역할 | 비고 |
|---|---|---|
| `shiftreg.c` | 74HC595 비트뱅잉 드라이버 | 이번에 신규 작성 |
| `dotmatrix.c` | 8x8 프레임버퍼 + Row 스캔 + 점멸 레이어 | 이번에 신규 작성 |
| `adc.c` | 조이스틱용 2채널 ADC | 이번에 신규 작성 (Project1엔 원래 없었음) |
| `joystick.c` | 조이스틱 X/Y/SW 읽기 | 이번에 신규 작성 |
| `main.c` | 미로 생성 + 게임 루프 + 부저 | 이번에 새로 작성 |
| `device_driver.h` | 위 신규 드라이버 extern 선언 추가 | 수정 |
| `exception.c` | `TIM4_IRQHandler`에서 `Dot_Matrix_Scan_ISR()` 호출 추가 | 수정 |
| `led.c`, `key.c`, `uart.c`, `timer.c`, `clock.c`, `systick.c` 등 | 기존 랩 실습 코드 그대로 | 미변경 |

원래 4x4 키 매트릭스(SZH-EK076)용으로 `keypad.c`를 만들었었지만, 조이스틱으로 방향 전환하면서 안 쓰기로 해서 삭제됨.

## 조정 가능한 파라미터

| 이름 | 위치 | 기본값 | 설명 |
|---|---|---|---|
| `JOY_MARGIN` | `main.c` | 600 | 중립점에서 이만큼 벗어나야 이동으로 인식 (클수록 둔감) |
| `BLINK_FRAMES` | `dotmatrix.c` | 10 | 이 프레임 수(1프레임=16ms)마다 점멸 토글 |
| 이동 후 딜레이 | `main.c` (`TIM2_Delay(300)`) | 300ms | 한 번 기울일 때 여러 칸 이동하는 것 방지용 디바운스 |
| Row 스캔 주기 | `dotmatrix.c` (`TIM4_Repeat_Interrupt_Enable(1, 2)`) | 2ms | 너무 느리면 깜빡임 눈에 띔 |

## 알려진 제한사항

- 미로가 8x8 원본 해상도를 그대로 셀로 쓰기 때문에 통로 폭은 항상 1칸이고, 가끔 아주 짧은 경로가 나올 수 있음 (칸 수를 줄여 통로+벽을 분리하는 4x4 셀 방식보다 난이도는 낮아질 수 있음)
- 저가형 조이스틱이라 개체마다 중립점 오차가 있어 전원을 켤 때 조이스틱을 건드리지 않은 상태여야 정확히 보정됨
